# Xiaohongshu Content Publish

## 适用场景

当用户要做小红书发布内容相关任务时使用本技能，包括：
- 从关键词或调研结果生成可发布内容
- 改写已有文案，提升点击率与互动率
- 输出多版本标题/正文/标签用于 AB 测试
- 生成评论区引导、发布节奏与复盘要点

## 输入要求

优先从用户或上游节点收集以下字段（缺失时先问最关键项）：
- 主题：本次内容核心话题
- 目标人群：年龄/城市/需求阶段
- 发布目标：涨粉 / 收藏 / 私信 / 转化
- 素材来源：笔记 JSON、用户草稿、卖点清单
- 约束：字数、语气、禁用词、品牌规则

最小可用输入：`主题 + 目标人群 + 发布目标`。

## 输出格式（默认）

始终输出以下 6 块，确保可直接发布：

1. 选题定位  
   - 一句话定位  
   - 核心利益点（3 条）

2. 标题候选（10 条）  
   - 前 5 条偏搜索流量  
   - 后 5 条偏点击吸引  
   - 避免夸张承诺和违规词

3. 正文（2 个版本）  
   - `版本A-干货清单型`（结构化、可收藏）  
   - `版本B-故事体验型`（更强代入感）  
   - 每个版本包含：开头钩子、主体、结尾行动引导

4. 标签与关键词  
   - 标签 8-12 个（核心词 + 长尾词）  
   - 给出 3 个主关键词用于后续追踪

5. 评论区引导（5 条）  
   - 引导用户留言、收藏、转发、私信

6. 发布建议  
   - 发布时间建议（工作日/周末）
   - 封面文案 3 条
   - 首图/配图建议 3 条

## 数据真实性约束（强制）

- 仅可基于输入提供的事实与抓取数据生成内容，不得虚构作者经历、地点细节、价格、数据指标、体验结论。  
- 若抓取数据中不存在某个用户要求的信息（如预算、交通、签证、拍照点），不要编造；该部分直接省略，或明确写“暂无数据支持”。  
- 不得把推测当事实输出；推测内容必须显式标注“推测/建议”。  
- 不得引用未在输入 `notes` 中出现的“热门帖子观点”。  
- 输出前自检：每个关键结论都能在输入数据中找到依据。

## 高赞风格迁移（强制）

- 允许参考高点赞帖子做“风格迁移”，目标是更像真人创作，但只迁移表达方式，不迁移具体句子。  
- 默认从输入 `notes` 中按 `like_count` 降序选前 3-6 条作为风格样本。  
- 先提炼样本的风格特征，再用于生成：
  - 开头方式（提问/反差/痛点直给）
  - 句长与节奏（短句密度、分段习惯）
  - 语气（克制种草/经验分享/清单导向）
  - 常见结构（先结论后细节、分点编号、结尾互动）
- 禁止复制样本中的原句、标题、专有表达；相似度过高时必须改写。  
- 生成后做一次“去 AI 味”检查：
  - 删除空泛套话（如“希望对你有帮助”重复出现）
  - 减少机械排比与模板化转折词
  - 增加基于数据可证据化的细节，而不是泛形容词

## 生成流程

1. 提炼输入信息，先写“用户真正关心什么”。  
2. 根据发布目标确定内容骨架：  
   - 涨粉：强调情绪共鸣 + 身份认同  
   - 收藏：强调清单化 + 可执行步骤  
   - 转化：强调对比、证据、行动路径  
3. 先产出标题，再写正文，最后补标签与评论引导。  
4. 检查合规性：  
   - 不写绝对化承诺（如“100%”“必涨粉”）  
   - 不写医疗/功效等高风险表述（如不确定就降级表达）  
5. 输出前做一次“平台友好改写”：短句、口语化、可扫读。

## 风格约束

- 默认中文简体，口语化但不低俗  
- 每段 1-3 句，优先短句  
- 关键收益点前置  
- 避免空洞形容词，多给具体场景/动作

```markdown
1.

```

## 当输入是抓取 JSON 时

如果输入包含 `notes` 数组：
- 先提炼高频主题、痛点词、情绪词  
- 再映射为“可发布语言”（避免直接复制他人原文）  
- 输出“差异化角度”至少 3 条，减少同质化
- 如果用户指定了必须覆盖的维度，仅在 `notes` 有证据时才输出该维度；无证据则不输出该维度内容。
- 额外输出“风格参考摘要”（内部使用，不写进最终帖子）：说明采用了哪些高赞帖子风格特征。

若 `notes` 超过 30 条，优先使用前 20 条 + 高频词统计，保证速度与稳定性。
