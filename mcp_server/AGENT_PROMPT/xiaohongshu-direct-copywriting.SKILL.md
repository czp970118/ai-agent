# Xiaohongshu Direct Copywriting

## 目标

在最少输入下，直接产出可发布的小红书内容，避免冗长分析。默认输出包含：
- 标题候选
- 正文成稿
- 标签
- 评论区引导

## 触发条件

当用户出现以下意图时直接启用：
- “直接写文案”
- “给我一篇小红书可发内容”
- “按这个主题出标题和正文”

## 输入要求

最小输入：
- 主题（必填）

可选输入：
- 目标人群（如：大学生、上班族、宝妈）
- 语气（如：真实分享、种草、干货清单）
- 目标（收藏/评论/私信/转化）
- 约束（字数、禁用词、品牌信息）

如果只给了主题，不追问，直接生成默认版本。

## 固定输出结构

始终按以下顺序输出：

1) 标题（6 条）
- 3 条偏搜索
- 3 条偏点击

2) 正文（1 篇可直接发布）
- 开头钩子（1-2 句）
- 主体内容（3-5 段，短句可扫读）
- 结尾互动引导（1 段）

3) 标签（8-12 个）
- 核心词 + 长尾词混合

4) 评论区引导（3 条）
- 引导留言 / 收藏 / 互动

## 写作规范

- 中文简体，口语化，避免AI腔
- 每段 1-3 句，优先短句
- 不写绝对化承诺（如“100%有效”）
- 不虚构未提供的事实、价格、经历、数据
- 不出现违规或高风险表达

## 默认风格（用户未指定时）

- 风格：真实经验 + 清单化表达
- 目标：优先提升收藏率
- 篇幅：正文约 350-600 字

## 快速模板

```markdown
【标题（6选1）】
1.
2.
3.
4.
5.
6.

【正文】
[开头钩子]
[主体分段]
[结尾互动引导]

【标签】
# # # # # # # #

【评论区引导】
1.
2.
3.
```

## 生成策略（内部）

1. 先定“用户收益点”（省钱/省时/少踩坑/变好看/效率提升）
2. 标题围绕收益点展开，不堆砌形容词
3. 正文优先“可执行动作”，减少空话
4. 输出前检查是否可直接复制发布
