# Xiaohongshu Travel Summary

## 目标

把小红书旅游帖子（`notes`）总结为可复用的“选题洞察 + 内容框架 + 发布策略”，用于：
- 后续生成旅游文案
- 复盘某个城市/主题的高频玩法
- 提炼稳定可复用的标题和标签模式

## 数据基线（来自当前缓存库）

当前在线缓存样本特征（`xhs_cache.online.db`）：
- 帖子总量：`44`
- 正文平均长度：约 `618` 字（最短 `12`，最长 `1107`）
- 高频标签：`攻略`、`广州塔`、`广州旅游`、`3天`、`三天`、`两天`、`深圳旅游`、`本地美食`
- 常见主题：广州/深圳城市旅游、`2-3天` 行程、景点+美食组合

> 结论：样本偏“城市周末游 + 攻略清单型”，总结时优先提炼“路线、天数、打卡点、吃住建议”。

## 输入要求

优先输入：
- `notes`: 抓取到的帖子数组（至少 8 条，建议 20 条）
- `topic`: 主题词（如“广州旅游”）
- `requirements`（可选）：如“两天/亲子/预算”

缺失 `notes` 时，不生成“数据结论”，只输出“待补充数据清单”。

## 输出结构（强制）

按以下 6 段输出，禁止省略标题：

1. **样本概览**
   - 样本量、主题覆盖、正文长度区间
   - 数据可信度判断（高/中/低）

2. **高频内容主题（Top 5）**
   - 每个主题 1 行：主题名 + 证据（来自哪些帖子特征）
   - 示例：`城市地标打卡（广州塔/天文台等反复出现）`

3. **可复用内容框架**
   - 给出 2 套框架：
     - `清单攻略型`（适合收藏）
     - `体验叙事型`（适合互动）
   - 每套包含：开头句型、主体结构、结尾引导

4. **标题模式总结**
   - 输出 5 条“标题模板”（不是具体标题）
   - 模板需贴合旅游样本风格（天数、路线、避坑、本地人视角）

5. **标签策略总结**
   - 核心标签（3-5）
   - 长尾标签（5-8）
   - 禁止堆砌重复语义标签（如“3天/三天”只保留一个写法）

6. **发布建议**
   - 推荐选题切入角度（3 条）
   - 下一轮采样建议（缺哪些数据）

## 写作约束（强制）

- 只基于输入 `notes` 说结论，不编造“爆款规律”
- 结论句后尽量带“证据短语”（如“多篇标题包含‘攻略/两天/三天’”）
- 语言简洁，避免空话（如“内容很优质”“值得参考”）
- 不要直接抄原文句子，输出应是总结后的抽象能力

## 旅游场景优先规则

当主题是城市旅游（广州/深圳等）时，优先提炼：
- `天数`（1日/2天/3天）
- `路线`（是否可地铁直达、是否步行友好）
- `景点分层`（必去/可选）
- `餐饮与预算提示`
- `避坑点`（排队、时段、人流）
